RED ELISAR PROMPT AND COMMAND CHECKLIST

Purpose:
Use this file as a ready-to-run checklist for validating the interactive Red Agent flow and report generation.

Prerequisites:
1. Start vulnerable app:
   python app.py

2. Activate virtual environment (project root):
   (Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned) ; (& "c:\mini project\.venv\Scripts\Activate.ps1")

3. Move to project root:
   Set-Location "c:\mini project"


CORE RUN COMMAND:
Run interactive CLI:
  "c:/mini project/.venv/Scripts/python.exe" run.py

VALIDATION COMMANDS FOR QUALITY

1. Count vulnerabilities in latest scan report:
$f = Get-ChildItem ".\red_agent\output" -Filter "scan_report_*.txt" | Sort-Object LastWriteTime -Descending | Select-Object -First 1 -ExpandProperty FullName
($content = Get-Content $f) | Select-String -Pattern "^\[VULNERABILITY #\d+\]" | Measure-Object

2. Check SQL mapping quality:
$content | Select-String -Pattern "Type: SQL Injection|Technique ID:" -Context 0,1

3. Check chain length per vulnerability section (manual quick check):
Look for numbered steps under ATTACK CHAIN and verify each chain is 5 to 6 steps.

4. Check that forms were discovered:
$j = Get-ChildItem ".\red_agent\output" -Filter "interactive_web_scan_*.json" | Sort-Object LastWriteTime -Descending | Select-Object -First 1 -ExpandProperty FullName
(Get-Content $j -Raw | ConvertFrom-Json).discovery.forms | Select-Object -First 10


PROMPT TEMPLATES FOR ITERATIVE TESTING

Prompt template: SQL injection focused
"Scan for SQL injection in search/login, map to MITRE with strict context, keep realistic web tools only."

Prompt template: data exposure focused
"Analyze exposed .env or backup resources and generate context-correct ATT&CK chain with no malware noise."

Prompt template: CORS and headers focused
"Validate CORS and missing security headers, map with primary technique accuracy and compact logical chain."


PASS CRITERIA

1. Duplicate endpoint and same vulnerability type are merged.
2. Primary technique is context-correct (for example SQL Injection to T1190, .env exposure to T1552).
3. Chain steps are 5 to 6, logical, and non-random.
4. Tool list is realistic and web-relevant.
5. Form-driven vulnerabilities are included in final report.
