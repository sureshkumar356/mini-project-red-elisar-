1b) ARCHITECTURE FOR LIVE VULNERABLE FINDING + RAG MAPPING
--------------------------------------------------------------------------------

Use this as the architecture slide text:

Input:
- Target URL (http://127.0.0.1:5000)
- Optional scenario + target environment metadata

Pipeline:
1. Discovery Layer
   - Crawl routes
   - Extract links, forms, actions, and parameters

2. Live Scanning Layer
   - Passive checks: headers, CORS, transport, exposure paths
   - Active checks: SQLi, XSS, auth bypass, redirect, cookie flags
   - Form probes: payload testing on discovered form inputs

3. Normalization + Dedup Layer
   - Merge key: endpoint path + normalized vulnerability type
   - Consolidate repeated evidence strings
   - Normalize severity + vulnerability family labels

4. RAG Intelligence Layer
   - Build contextual query from vulnerability + endpoint + evidence
   - Retrieve MITRE ATT&CK candidates from FAISS index
   - Apply context filtering and relevance threshold
   - Enforce primary mapping by vulnerability family
     - SQLi -> T1190
     - Exposed credentials/.env -> T1552
     - Plain HTTP/weak transport -> T1048

5. Attack Chain Construction Layer
   - Generate 5 to 6 step chain with realistic tactic order
   - Reject unrelated tactic jumps and noisy software/tool suggestions

6. Reporting Layer
   - interactive_web_scan JSON/MD
   - vuln_report JSON/MD
   - final analyst TXT report

One-line architecture message for PPT:
"Red ELISAR converts live vulnerable web findings into deduplicated, context-grounded ATT&CK chains using retrieval-constrained generation."
